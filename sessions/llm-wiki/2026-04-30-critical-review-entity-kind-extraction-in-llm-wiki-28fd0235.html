The entity-kind pipeline at a glance

Documents land in data/. They flow through:

  1. llm_wiki/project.py::ProjectWiki.compile() — orchestrates the run, calls the batch extractor.
  2. llm_wiki/batch.py::BatchIngestRunner.run() — feeds each markdown file to a ResearchGraphExtractor.
  3. llm_wiki/research_graph.py::ResearchGraphExtractor.extract_text() — the heart. Decides what node types appear. Key helpers:
  4. source_kind_to_node_type(source_kind, source_path) — picks Paper / Repository / SourceDocument from the path.
  5. _add_document_structure(builder, document, text, ...) — heading → optional Concept.
  6. _extract_paper_references(builder, document, text, ...) — body URL → Paper node, with infer_arxiv_reference_title() filtering noise.
  7. looks_like_research_title() / is_verified_paper_title() — gates.
  8. is_public_research_node() — gates whether a node ever appears on the public website.
  9. llm_wiki/wiki_projector.py::WikiLayerProjector.project() — projects each public node into a wiki/<kind>/<slug>.md page.
  10. llm_wiki/synthesis.py::SynthesisProjector.project() — builds higher-order syntheses, with its own title_quality filter.
  11. llm_wiki/site/search.py and pages.py — render the website. Recent changes filter to public nodes only.
